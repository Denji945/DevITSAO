2) Presentación

La materia se centra en el desarrollo web completo (Full Stack): arquitectura cliente-servidor, desarrollo front-end accesible y responsivo, uso de frameworks modernos, patrones de diseño en back-end, Angular, servicios RESTful, integración con Firebase, y prácticas de control de versiones e integración/despliegue continuo para construir soluciones escalables.

Intención didáctica

Que el estudiante comprenda y aplique conceptos, herramientas y patrones para construir, implementar y mantener aplicaciones web accesibles, responsivas y escalables, integrando front-end, back-end y buenas prácticas de versionado e integración continua.

4) Competencia específica

Diseñar e implementar interfaces web accesibles, responsivas y usables con tecnologías/frameworks actuales (HTML5, CSS3, lenguajes de servidor como PHP/JSP/ASP, Bootstrap y Angular), y aplicar patrones de diseño en la lógica del servidor para construir arquitecturas sólidas, escalables y mantenibles.

6) Temario

1. Fundamentos del Desarrollo Full Stack
  • Rol y diferencias: Front-End, Back-End y Full Stack.
  • Arquitectura cliente-servidor y enfoque en capas.
  • Navegadores, servidores web, hosting y dominios.
  • Herramientas: Node.js, NPM, Git y VS Code.
  • Control de versiones con Git/GitHub y CI/CD con GitHub Actions.
2. Interfaces Web Modernas y Accesibles
  • HTML5 y estructura semántica.
  • CSS3: modelo de caja y media queries.
  • Bootstrap/Tailwind: componentes reutilizables.
  • Accesibilidad (WCAG) y diseño responsive/adaptativo.
3. Patrones de Diseño para la Arquitectura del Back-End
  • Creacionales: Singleton, Factory Method.
  • Estructurales: Adapter, Decorator, Composite.
  • Comportamiento: Observer, State.
4. Desarrollo Front-End con Frameworks (Angular)
  • Introducción a Angular y Angular CLI (componentes, módulos, servicios).
  • JSON y métodos HTTP (GET, POST, PUT, DELETE).
  • Consumo/creación de servicios con async/await.
  • Firebase y BaaS.
  • Formularios reactivos, validaciones, ruteo y navegación.

8) Prácticas sugeridas

  • Instalar Node.js, Git y Visual Studio Code.
  • Crear una página de presentación con etiquetas semánticas.
  • Menú adaptable (Tailwind o Bootstrap).
  • Mejoras de accesibilidad: ARIA, alt, label y navegación con teclado.
  • Factory Method: createUser(type) (Admin, Cliente, Invitado).
  • Adapter para integrar una API simulada.
  • Decorator para añadir estilos/funcionalidades a un objeto.
  • Angular CLI: al menos 3 componentes + un servicio para compartir datos.
  • Consumir API pública (ej. JSONPlaceholder) con GET/POST/PUT/DELETE.
  • Formulario reactivo con validaciones (nombre, email, contraseña).
  • Ruteo entre vistas (inicio, formulario, dashboard).

9) Proyecto de asignatura (fases)

El proyecto busca demostrar el desarrollo y alcance de las competencias de la materia mediante:

Fundamentación

Marco referencial (teórico, conceptual, contextual y legal) y diagnóstico del problema/situación.

Planeación

Diseño del proyecto, actividades, recursos y cronograma (con asesoría docente).

Ejecución

Implementación del proyecto; fase de mayor duración y desarrollo de competencias.

Evaluación

Juicio de valor: logros y mejoras; enfoque de mejora continua y pensamiento crítico.
